Идея: Exam Prep Bot для аналитической геометрии и линейной алгебры
Оценка по критериям
Критерий	Оценка
Простой в разработке	✅ Да — CRUD + LLM для проверки ответов
Очевидно полезный	✅ Да — студенты готовятся к экзамену
Легко объяснить	✅ Да — “бот даёт задачи, проверяет ответы, показывает теорию”
Определение продукта
Элемент	Описание
End-user	Студент, готовящийся к экзамену по аналитической геометрии и линейной алгебре
Problem	Нет удобного способа тренироваться: задачи разбросаны, нет мгновенной проверки, теорию нужно искать отдельно
One sentence	Веб-приложение, которое даёт задачи по темам, проверяет ответы через LLM и показывает теорию по запросу
Core feature	Студент решает задачу → LLM проверяет ответ и объясняет ошибки → по запросу показывает теорию
Что нужно построить
Компонент	Что делаем	Используем готовое или с нуля?
Database	Таблицы: topics, tasks, theory, attempts	PostgreSQL (готовый), но новая схема
Backend	API: получить задачу, отправить ответ, получить теорию	FastAPI (готовый фреймворк), но новые endpoints
Client	Чат-интерфейс: бот даёт задачу → студент отвечает → LLM проверяет	Flutter + Nanobot (готовые), новый промпт + MCP-инструмент
Version 1 (одно ядро)
Core feature: Студент выбирает тему → получает задачу → отвечает → LLM проверяет и объясняет

База с 10-15 задачами по 2-3 темам (скаляры, векторы, матрицы)
Backend: GET /task/{topic}, POST /check, GET /theory/{topic}
Nanobot с MCP-инструментом get_task, check_answer, show_theory
Flutter-чат: студент пишет “дай задачу по матрицам” → получает задачу → пишет ответ → LLM проверяет
Version 2 (улучшения)
Прогресс: какие задачи решены, какие — нет
LLM объясняет ошибки подробнее с ссылками на теорию
Режим “экзамен”: случайные задачи из всех тем
Deploy и доступ по ссылке
Отличие от готового LMS
Готовый LMS	Наш продукт
Аналитика по студентам, дедлайнам	Тренировка к экзамену, проверка ответов
MCP-инструменты: lms_learners, lms_pass_rates	MCP-инструменты: get_task, check_answer, show_theory
Дашборд преподавателя	Чат-бот студента
Вывод: инфраструктуру используем (Docker, FastAPI, PostgreSQL, Nanobot, Flutter), но содержимое своё — новая схема БД, новые endpoints, новые MCP-инструменты.
